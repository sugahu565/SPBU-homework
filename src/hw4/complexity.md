Сложность переборного решения:
Генерация всех перестановок через permutations по сложности О(n * n!) (размер кортежа для генерации * на все возможные перестановки)
Для каждой перестановки идёт n * (n - 1) / 2 операций - для каждой пары ферзей проверяется, не мешают ли они друг другу.
Количество операций: n * n! * n * (n - 1) / 2 = O(n^3 * n!)
Итоговая сложность: O(n^3 * n!)


Сложность рекурсивного решения:
Алгоритм начинается с перебора n вариантов расстановки на 1 строку. Пусть подошло n - k вариантов
Далее начинаются n - k рекурсий, в каждой проверяется n вариантов. Пусть в среднем подошло n - k - m вариантов в каждой
...
И так до самого конца
Общее число операций: n * (n - k) * (n - k - m)
Мы никак не можем сказать, чему равно значение n - k, n - k - m..., лишь только то, что оно дальше идёт меньше n
Итоговая сложность: О(n!)
